# EGDESK SMS 시스템 공식 매뉴얼

안녕하세요! 이 문서는 EGDESK SMS (평생 무료 문자 발송 시스템)의 주요 기능과 사용법을 설명하는 공식 가이드입니다. 

## 1. 대시보드
대시보드에서는 시스템의 전반적인 상태를 한눈에 확인할 수 있습니다.
- **최근 실적 요약**: 문자 발송 건수, 성공/실패 현황.
- **주요 통계 그래프**: 주간, 월간 비즈니스 지표 요약.
- **시스템 상태**: AI 연동 상태, 데이터베이스 연결 상태.

## 2. 메시지 및 문자 관리
EGDESK의 핵심 기능인 무료 문자 발송 및 관리 메뉴입니다.
- **무료 문자 발송 AI**: 개별 고객 또는 단체 고객을 선택하여 즉시 문자를 발송할 수 있습니다. 템플릿(자주 쓰는 문구)을 활용하면 편리합니다.
- **발송 내역 조회**: 발송된 문자의 성공, 실패(에러 사유), 예약 대기 상태를 실시간으로 추적합니다.
- **자동 발송 설정**: 특정 조건(예: 회원가입 시, 예약 확정 시, 주문 완료 시)이 충족되면 AI가 자동으로 안내 문자를 발송하도록 시나리오를 설정합니다.

## 3. 고객 및 비즈니스 관리
고객의 생애 주기를 관리하고 마케팅 효율을 높이는 기능들입니다.
- **고객 관리**: 고객의 기본 정보, 연락처, 방문 기록 등을 관리하며, VIP 고객을 분류하여 타겟 마케팅을 진행할 수 있습니다.
- **쿠폰 관리**: 새로운 할인 쿠폰을 생성하여 발급하고, 유효기간 및 특정 상품/카테고리별 사용 제한 조건을 한눈에 구성하고 통제합니다.
- **예약 관리**: 식당, 미용실 등 예약이 필요한 업종에서 달력 형태로 일정을 관리합니다.
- **주문내역 관리 / 결제내역 관리**: 고객의 상품 구매 내역 및 결제(카드, 포인트 등) 상태를 확인하고 환불 처리를 지원합니다.

## 4. 시스템 설정 (AI 비서 연동)
- **AI 비서 및 이지봇 모델 설정**: 구글 AI(Gemini) API 키를 등록하여 이지봇(EasyBot)을 활성화합니다.
- API 키가 정상적으로 등록되어야 이지봇이 로컬 저장소와 데이터베이스를 분석하여 인공지능 답변을 생성할 수 있습니다.

> **💡 이지봇(EasyBot) 활용 팁**
> 이지봇에게 "최근 1주일간 가장 많이 팔린 메뉴 알려줘" 또는 "문자 발송 실패 건수가 몇 개야?"라고 물어보시면, 
> 복잡한 화면 이동 없이 즉석에서 데이터베이스를 조회하여 결과를 알려줍니다!

## 5. 음성 대화 기능 (Voice UI)
이지봇은 키보드 타이핑 없이 목소리로 대화할 수 있는 완벽한 **음성 비서 기능(STT/TTS)**을 제공합니다.
- **음성으로 질문하기(STT)**: 챗봇 창 아래쪽 마이크(🎙️) 버튼을 누르고 말씀하세요. 인식이 끝나면 자동으로 텍스트로 변환되어 질문이 전송됩니다.
- **음성으로 대답 듣기(TTS)**: 챗봇 창 상단 헤더의 스피커(🔊) 버튼이 활성화되어 있으면, 이지봇이 특수문자를 제외하고 친절한 한국어 음성으로 대답을 읽어줍니다. 끄고 싶을 때는 스피커 버튼을 다시 눌러 비활성화할 수 있습니다.

## 6. 고도화된 UI/UX 편의 기능 (검색 & 페이지네이션)
더욱 직관적이고 쾌적한 매장 관리를 위해 시스템 내 13대 핵심 관리 대장 화면에 강력한 검색 및 페이지네이션 UX가 일괄 적용되었습니다.
- **실시간 다중 통합 검색**: 이름, 연락처, 태그/그룹명을 검색창에 입력하면 엔터 키를 누를 필요 없이 입력 즉시 실시간으로 대상이 필터링됩니다.
- **페이지당 표시 개수 제어**: 각 화면 하단에서 데이터를 5개/10개/20개/50개/100개씩 나누어 볼 수 있는 조절 셀렉터를 탑재하여 대량의 데이터도 버벅임 없이 쾌적하게 로딩합니다.
- **안정적인 UI/UX 상시 노출**: 검색 결과가 없거나 전체 데이터가 1페이지만 존재하는 경우에도 페이지네이션 바가 사라지지 않고 항상 비활성화(disabled)된 상태로 하단에 유지되므로, 일관성 있고 안정감 있는 관리 환경을 제공합니다.
- **[핵심 💡] 단체 문자 발송 시 다중 페이지 체크 누적 보존**: 문자 발송 페이지에서 대상을 선택할 때, 페이지를 이동하거나 표시 개수를 변경하더라도 **이전에 체크해 둔 고객들의 선택 상태가 그대로 유지(캐싱)**됩니다. 이를 통해 수백 명의 타겟 고객을 실수나 누락 없이 안전하게 한 번에 지정하여 단체 문자를 발송하실 수 있습니다.

## 7. 쿠폰 유효기간 및 사용 제한(예외 처리) 기능
쿠폰의 남용을 막고 점주님의 상품 마진율을 정교하게 방어하기 위해 **유효기간 설정 및 초고속 인메모리(RAM) 캐시 기반의 상품별 사용 제한 기능**이 탑재되었습니다.

### 7.1. 쿠폰 유효기간(만료일) 관리
- **유효기간 캘린더 피커**: 쿠폰 발행 시 만료일을 달력으로 손쉽게 지정할 수 있습니다 (지정하지 않을 시 '무제한').
- **만료 쿠폰 자동 통제**: 유효기간이 지난 쿠폰은 상태 뱃지에 빨간색 **`[만료됨]`** 표시와 함께 행 전체에 **취소선 및 회색 비활성화 효과**가 자동으로 적용되며, 고객이 결제창에서 입력해도 자동으로 실시간 검증 차단됩니다.

### 7.2. 상품/카테고리별 사용 제한 (예외 처리)
- **적용 및 제한 설정**: 쿠폰 발행 화면 하단에서 적용 방식(할인 대상에서 제외하는 Blacklist / 할인 대상에만 허용하는 Whitelist)과 제한 종류(개별 상품 ID / 특정 카테고리명)를 조합하여 무제한으로 조건을 추가할 수 있습니다.
- **토큰 뱃지 UI**: 추가된 조건은 직관적인 빨간색(제외)/남색(허용) 태그 배지 형태로 나열되어 오설정을 줄여줍니다.
- **테이블 요약 뱃지**: 발행된 쿠폰 목록에서 각 쿠폰에 어떤 예외 조항이 걸려 있는지 요약 뱃지(예: `제외:상품(prod-1)`)를 한 눈에 파악할 수 있습니다.

### 7.3. 상품 DB 내 쿠폰 일괄 제어 토글 (마진율 실시간 방어)
- **원클릭 On/Off 스위치**: '상품 DB 관리' 화면에 '쿠폰 적용' 열을 신설하여, 점주가 각 상품별로 할인을 허용할지 제외할지 버튼 클릭 한 번으로 즉시 토글할 수 있습니다.
- **초고속 인메모리(RAM) 캐시 엔진 연동**: 외부 설치가 필요한 Redis 대신 Next.js 서버 메모리(RAM)를 100% 활용하는 경량 캐시 엔진을 탑재하여, 대량의 트래픽이 몰리더라도 **데이터베이스(DB) 병목 없이 수 마이크로초(μs) 대의 무마찰 가격 검증**을 처리합니다.
- **제외 시 고객 피드백**: 소비자가 장바구니에 쿠폰 적용 불가능 상품을 담았을 때, 단순히 먹통이 되지 않고 *"제한 품목을 제외한 적용 가능 금액이 최소 주문 조건을 만족해야 합니다"*와 같은 상세하고 친절한 안내를 붉은색 텍스트로 즉각 연동하여 이탈을 예방합니다.
